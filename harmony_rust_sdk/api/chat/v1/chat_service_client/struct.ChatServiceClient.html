pub fn create_direct_message<Req>(
    &mut self,
    req: Req
) -> impl Future<Output = ClientResult<Response<CreateDirectMessageResponse>, InnerErr>> + 'static where
    Req: IntoRequest<CreateDirectMessageRequest>, 

Endpoint to create a “direct message” guild.

  • The inviter and the invitee that join the created guild will both be owners.
  • The guild should be created on the server that inviter is on.
  • The server should send a guild invite to the invitee (specified in the request), using the UserInvited postbox event if the invitee is on another server, otherwise see the below item.
  • The server should process this as follows: adding the invite to their pending invite list and sending an InviteReceived event over their event stream if the invitee is on this server.
  • The invitee may or may not use the invite. Only the invitee may use the invite.

pub fn reject_pending_invite<Req>(
    &mut self,
    req: Req
) -> impl Future<Output = ClientResult<Response<RejectPendingInviteResponse>, InnerErr>> + 'static where
    Req: IntoRequest<RejectPendingInviteRequest>, 

Endpoint to reject (delete with notification to the inviter) an invite from your pending invite list.

If the invitee is on a different server than the inviter, the invitee’s server should send a UserRejectedInvite postbox event to the inviter’s server.

The “notification” is sending a InviteRejected stream event to the inviter. If the guild’s kind is DirectMessage, the server should also set the rejected field of the guild’s kind to true.
